GrokBridge 配置页

这个页面现在支持真正的多账号模式。你可以把多个 Grok 账号会话导入为一个列表，运行时会按列表顺序轮询使用。页面不会直接读取你本机浏览器 Cookie，需要你从浏览器复制 `Request Headers`、`Copy as cURL` 或整段 Cookie 后再导入。

第 1 步：导入账号 在你自己的浏览器登录 `grok.com`，复制一个请求的 `Request Headers` 或 `Copy as cURL`，然后粘贴到下方导入区。
第 2 步：整理账号列表 每导入一个账号，就把它加入账号列表。多个账号可以编辑、删除、调整顺序。
第 3 步：保存并调用 填写调用 API Key，保存全部配置后，用右侧动态显示的 `/v1` 地址开始调用。

STEP 3

调用 API Key

这是调用当前 GrokBridge `/v1` OpenAI 兼容接口时使用的 Bearer 密钥，不是 Grok 官网账号密码。它是全局的，所有账号共用同一个调用 API Key。

客户端调用时，把请求发到当前页面对应的 `/v1` 地址。
保存时会把当前账号列表写入 `sessions` 数组。运行时请求会按列表顺序轮询：账号 1 -> 账号 2 -> 账号 3。
客户端调用上面的 API 地址时，把这个值放到 `Authorization: Bearer <key>` 里。若服务直接暴露公网，请在反向代理层再加一层访问控制。
